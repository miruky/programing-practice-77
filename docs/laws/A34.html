法則 A34

Grundy 数 — 独立ゲームを Nim に変換する

各部分ゲームの Grundy 数の XOR が 0 なら後手必勝

Nim 以外のゲームを 「Grundy 数」 という不変量に変換すると、全部分ゲームの Grundy 数の XOR が 0 のとき後手必勝 というNim と同じ判定法で勝敗が分かる、強力な理論。

Grundy 数の定義: 「遷移先の Grundy 数の集合に含まれない最小の非負整数 (mex)」。再帰的に計算でき、終端状態は Grundy 数 0。小さい N で計算して 周期性を発見 することがよくあります。

Step 1: mex の計算

mex(S) = S に含まれない最小の非負整数。mex({0,1,3}) = 2、mex({1,2}) = 0、mex({}) = 0。

flowchart LR
  S["{0, 1, 3}"] --> M[mex = 2]
  T["{1, 2}"] --> N[mex = 0]

Step 2: 1 山だけの Grundy 数

「石が a 個あって、1〜k 個取れる」なら Grundy(a) = a mod (k+1)。周期 (k+1) で繰り返す。より複雑な取り方は 後退解析 + mex で 1 つずつ計算する。

flowchart LR
  G0["G(0)=0"] --> G1["G(1)=1"]
  G1 --> G2["G(2)=2"]
  G2 --> G3["G(3)=0"]
  G3 --> G4["G(4)=1"]

Step 3: 複数山の合成

「N 個の独立ゲームの全体」の Grundy 数は 各ゲームの Grundy 数の XOR。XOR が 0 なら後手必勝、それ以外は先手必勝。

詰まりやすい点

  • 小さい N で 手計算 / プログラム計算で表を作り、周期を発見 するのが定石。
  • 「独立な部分ゲームに分解できる」ことが前提。盤面が絡み合うゲームには使えない。
  • Grundy 数を計算するだけのプログラムを別に用意して周期を見るとデバッグが楽。
