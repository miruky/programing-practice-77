法則 A51

スタック — 後入れ先出し (LIFO)

最後に入れたものを最初に取り出す

後入れ先出し (LIFO) のデータ構造。「最後に入れたもの」を最初に取り出すという挙動。Python では list の append / pop か、collections.deque の append / pop で実現。

関数の呼び出し履歴、DFS、括弧マッチング、Histogram の最大長方形 など、「後で戻ってきて処理する」性質の問題で頻出。

Step 1: 操作

  • push: 末尾に追加 (stack.append(x))
  • pop: 末尾を取り出す (stack.pop())
  • top: 末尾を見る (stack[-1])

全て O(1)。

flowchart LR
  E[空] -->|push 1| S1[1]
  S1 -->|push 2| S2["1, 2"]
  S2 -->|push 3| S3["1, 2, 3"]
  S3 -->|pop = 3| S4["1, 2"]

Step 2: list vs deque

Python の list も末尾操作は O(1) なのでスタックとして使える。deque の方がメモリ効率が良いが、競プロでは list で十分。

Step 3: DFS のスタック実装

再帰 DFS を 明示的なスタック で書けば、再帰上限を気にせず深い探索ができる。

stack = [start]
while stack:
    v = stack.pop()
    for u in G[v]:
        if not visited[u]:
            visited[u] = True
            stack.append(u)

詰まりやすい点

  • 空のスタックで pop() は IndexError。if stack: でガード。
  • deque の場合は append / pop、popleft は使わない (使うとキューになる)。
  • 再帰 DFS のスタック上限は Python 1000。sys.setrecursionlimit(10**6) で拡張可。
