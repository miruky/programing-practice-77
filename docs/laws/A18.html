法則 A18

部分和問題 — 「i 枚から合計 j 作れるか」の bool DP

数値を作れるかを True/False で広げていく

N 枚のカードからいくつか選んで合計を S にできるか? を判定する典型問題。dp[i][j] = i 枚目までから合計 j を作れるか という 2 次元 bool DP で解けます。

「使う / 使わない」の 2 択 で全探索しても 2^N (= N=30 で 10^9)。DP に書き直すと O(NS) で、N=100, S=10^4 でも瞬時。「DP は全探索の言い換え」 という感覚を掴むのに最適な問題です。

Step 1: 状態の定義

dp[i][j] = 最初の i 枚から合計 j を作れるか (True / False)。状態数は (N+1) × (S+1) ≒ 10^6 まで現実的。

flowchart LR
  D["dp[i][j]"] --> Q[True / False]
  Q -.意味.- M["i 枚目までで合計 j が作れる?"]

Step 2: 初期状態

dp[0][0] = True (0 枚から合計 0 は空和で作れる)、dp[0][1..S] = False (0 枚で正の合計は不可能)。

Step 3: 遷移

i 枚目のカードの値 A[i-1] について、「使わない」と「使う」の OR。

  • 使わない: dp[i][j] = dp[i-1][j]
  • 使う (j >= A[i-1]): dp[i][j] |= dp[i-1][j - A[i-1]]
flowchart LR
  P1["dp[i-1][j] (使わない)"] --> OR[OR]
  P2["dp[i-1][j-A[i-1]] (使う)"] --> OR
  OR --> D["dp[i][j]"]

詰まりやすい点

  • 「使う」遷移は j >= A[i-1] のときだけ。条件分岐を忘れない。
  • dp[N][S] を見て True なら Yes、False なら No。
  • 値の総和が大きいとメモリが厳しい。1 次元に圧縮する別解もある (dp[j] |= dp[j - A[i-1]])。
