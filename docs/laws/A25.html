法則 A25

経路数え上げ — 2 次元 DP で数える典型

格子上の経路数。障害物を `0` で消すだけ

H × W の格子で、左上から右下まで「右」「下」だけで移動する経路数を求める。dp[i][j] = (0,0) から (i,j) に来る経路数 という最も素直な 2 次元 DP。

遷移は dp[i][j] = dp[i-1][j] + dp[i][j-1]。障害物 # のマスは dp = 0 のまま残せばよく、「通れないなら 0 になる」という構造の美しさが光る。

Step 1: スタート地点を 1

dp[0][0] = 1 (スタート地点に来る方法は 1 通り、つまり最初からそこにいる)。他は 0 で初期化しておけば、障害物の処理が「触らない」だけで済む。

Step 2: 遷移

「上から来る」+「左から来る」の和。

dp[i][j] = dp[i-1][j] + dp[i][j-1] (障害物なら 0 のまま)。

flowchart LR
  U["dp[i-1][j] (上から)"] --> P[+]
  L["dp[i][j-1] (左から)"] --> P
  P --> D["dp[i][j]"]

Step 3: 障害物 / 端の処理

障害物 # のマスは初期値 0 のまま残す。端 (i=0 or j=0) は番兵を 0 で取っておけば dp[-1][j] などの参照を気にしなくてよい。

詰まりやすい点

  • 経路数は mod 10^9+7 で答えさせる場合が多い。最初に modulo の必要性を確認。
  • スタートまたはゴールが障害物の問題も稀にある。dp[0][0] の初期化を慎重に。
  • メモリが厳しいなら 1 行分の配列で in-place 更新できる (典型 1D 圧縮)。
