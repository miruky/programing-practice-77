法則 A77

切り方の最大化 — 「最小区間長 ≥ x」で二分探索

A12 と同じ枠組み。OK/NG の境目を最大化

「N 個の印から K 個選んで棒を切り、できる N+1 個の区間の最小長を最大化」という古典問題。「最小区間長 ≥ x にできるか?」 という判定問題に書き換えれば、x について NG → OK が単調 に変わるので二分探索。

判定は 左から見て「直前の切れ目から x 以上離れていれば切る」 の貪欲。判定 + 二分探索 = 「答えで二分探索」(A12) の典型パターン。

Step 1: 判定関数 check(x)

「最小区間長 ≥ x にできるか?」を判定。左から貪欲: last_kireme から x 以上離れていれば切る、を繰り返す。 切れ目の数が K 以上なら OK。

flowchart TB
  S[直前の切れ目 last] --> I[次の印 A i]
  I --> C{A i - last >= x AND L - A i >= x?}
  C -- Yes --> A[切る; last = A i; cnt += 1]
  C -- No --> SK[スキップ]

Step 2: 二分探索の方向

答えを最大化したいので 切り上げ版 の二分探索を使う。

while left < right:
    mid = (left + right + 1) // 2  # 切り上げ
    if check(mid):
        left = mid
    else:
        right = mid - 1

Step 3: 範囲

left = 1, right = L。「答え x は 1 〜 L の間にある」と分かっているので、その範囲で二分。

詰まりやすい点

  • 最大化のときは mid = (l + r + 1) // 2 で切り上げ。 切り上げ忘れで無限ループ。
  • 「最小値を最大化」「最大値を最小化」は両方とも答えで二分探索の出番。
  • 判定関数 check(x) の貪欲が本当に正しいか手で確認。
