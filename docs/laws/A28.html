法則 A28

mod 計算 — 毎回余りを取って小さく保つ

加減乗は mod の中でも成り立つ。除算だけは別扱い

答えが指数的に大きくなる問題で 「mod 10^9+7 で出力せよ」 と言われる典型。(a + b) % M = ((a % M) + (b % M)) % M などの性質を使い、毎回 mod を取って値を小さく保ちます。

C 言語ならオーバーフロー対策、Python なら高速化の目的が大きい。除算だけは mod 逆元 (A30) が必要になる別扱いの操作。

Step 1: 加減乗は mod の中で OK

(a ± b) % M = ((a % M) ± (b % M)) % M

(a × b) % M = ((a % M) × (b % M)) % M

計算の途中で % M を挟んでも結果は変わらない。

flowchart LR
  A[a] --> P["× b"]
  P --> M["% M"]
  M -.同じ.- A2["(a % M) × (b % M) % M"]

Step 2: 引き算の符号

(a - b) % M は b > a のとき負になる。Python は 負の値でも 0 以上の余り を返すので OK、C/C++ では + M を入れて補正する流儀。

Step 3: 除算は別法則

通常の (a // b) % M は成り立たない。mod 逆元 (A30) を使って a * inv(b) % M と書き直す。

詰まりやすい点

  • 加減乗の mod は 掛け算の前後 に挟むだけで OK。
  • 引き算で負になる Python と C/C++ の挙動の違いを意識。
  • 「mod 10^9+7」は素数なので、フェルマーの小定理で逆元が作れる (A30 で活用)。
