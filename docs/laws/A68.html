法則 A68

最大フロー — Ford-Fulkerson 法

s から t まで増加路を見つけて流せるだけ流す

ネットワークフロー: 始点 s からシンク t まで、各辺の 容量 を超えない範囲で流せる最大流量を求める。

Ford-Fulkerson: DFS で「まだ流せる路 (増加路)」を見つけ、流せるだけ流す。増加路がなくなったら終了。残余グラフと逆辺 の概念がカギ。逆辺は 「流れを取り消せる余地」 を表す。

Step 1: 残余グラフの構築

各辺 (u→v, 容量 c) に対し、逆辺 (v→u, 容量 0) をペアで追加。後で流れを差し戻せるようにするための準備。

flowchart LR
  F["u→v 容量 c"] --> G[グラフ]
  B["v→u 容量 0 (逆辺)"] --> G

Step 2: 増加路探索

DFS で s → t のパスを探す。残容量 > 0 の辺だけ通る。見つかったら、パスの最小容量だけ流す。

flowchart LR
  S[s] --> A[a] --> B[b] --> T[t]
  S -.最小容量 F.- T

Step 3: 残余の更新

流した量 F を、順辺から引き、逆辺に足す。逆辺が増えると「もし良ければ後で取り消せる」状態になり、最適性が担保される。

flowchart LR
  N["順辺 -= F"]
  R["逆辺 += F"]

詰まりやすい点

  • 逆辺を入れ忘れると 最適でない。Ford-Fulkerson の本質。
  • 計算量は O(F·E)。F が大きいと遅い。Dinic 法 (BFS で短い path から) ならもっと速い。
  • 容量が浮動小数のときは収束しないことがある。整数限定が安全。
