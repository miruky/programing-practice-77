法則 A61

グラフの表現 — 隣接リストが基本

頂点ごとに「繋がっている頂点のリスト」を持つ

グラフをプログラムで扱う方法は 隣接行列 と 隣接リスト の 2 通り。競プロでは 隣接リストが標準。空間 O(V+E)、辺の走査が O(degree)。

G[v] = [v に繋がっている頂点のリスト] で表す。無向グラフは双方向に登録、有向は片方向のみ。重み付きなら (隣接頂点, 重み) のタプルを入れる。

Step 1: 隣接リスト

G = [[] for _ in range(N + 1)]
for _ in range(M):
    a, b = map(int, input().split())
    G[a].append(b)
    G[b].append(a)  # 無向なら両方
flowchart LR
  G1["G[1] = [2, 3]"] -.- N1["1 → 2, 1 → 3"]
  G2["G[2] = [1, 3, 4]"] -.- N2["2 → 1, 3, 4"]

Step 2: 隣接行列との比較

隣接行列 M[i][j] = 1 if edge else 0 は密グラフ向き、空間 O(V²)。隣接リスト は疎グラフ向き、空間 O(V+E)。競プロは疎が多いのでリストが多数派。

flowchart LR
  L[隣接リスト<br/>空間 O V+E] --> S[疎グラフに最適]
  M[隣接行列<br/>空間 O V^2] --> D[密グラフに最適]

Step 3: 重み付きグラフ

G[v].append((u, w)) でタプルにする。ダイクストラ (A64) や MST (A67) はこの形式が必須。

詰まりやすい点

  • 1-indexed の問題が多い。G = [[] for _ in range(N + 1)] で 1 個余分に取る。
  • 無向グラフで G[b].append(a) を忘れると一方通行に。
  • 多重辺・自己ループの扱いは問題による。問題文を確認。
