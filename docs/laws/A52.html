法則 A52

キュー — 先入れ先出し (FIFO)

入った順に取り出す。BFS の中核

先入れ先出し (FIFO) のデータ構造。最初に入れたものを最初に取り出す。Python では collections.deque を使う (list は popleft が O(N) で遅い)。

BFS、シミュレーション、生産消費 などで使う。両端から O(1) で操作できる deque は、スタック/キューの両方の役割をこなせる優秀なデータ構造。

Step 1: 操作

  • enqueue: 末尾に追加 (q.append(x))
  • dequeue: 先頭を取り出す (q.popleft())
  • front: 先頭を見る (q[0])

全て O(1)。

flowchart LR
  E[空] -->|append 1| Q1[1]
  Q1 -->|append 2| Q2["1, 2"]
  Q2 -->|append 3| Q3["1, 2, 3"]
  Q3 -->|popleft = 1| Q4["2, 3"]

Step 2: BFS の典型実装

from collections import deque
q = deque([start])
dist[start] = 0
while q:
    v = q.popleft()
    for u in G[v]:
        if dist[u] == -1:
            dist[u] = dist[v] + 1
            q.append(u)

Step 3: deque は両端 O(1)

append, pop, appendleft, popleft 全て O(1)。両端から取り出せる デック (double-ended queue) として、0-1 BFS や C17 のような特殊用途にも便利。

詰まりやすい点

  • list.pop(0) は O(N) で遅い。必ず deque.popleft() を使う。
  • 空のときの popleft() は IndexError。
  • queue モジュール (Queue 標準ライブラリ) はスレッド安全な代わりに遅い。競プロでは deque。
