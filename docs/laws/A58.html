法則 A58

セグメント木 (RMQ) — 区間最大値を O(log N) で

配列を木に変換。任意区間の集約と 1 点更新が log N

配列の 任意区間の最大値 を O(log N) で求め、1 点更新 も O(log N) で行えるデータ構造。「完全二分木で、葉が元の配列、親が子 2 つの max」という単純な構造。

max を sum に取り換えれば RSQ (A59)、min / GCD / XOR など、結合則 + 単位元 を持つ任意の演算で同じ枠組みが使える。「区間更新 + 区間クエリ」が欲しい場合は 遅延セグメント木 に拡張。

Step 1: 木の構造

N 以上の最小の 2 のべき乗を size とし、配列サイズ 2·size の配列で完全二分木を表現。葉は size..2·size-1、親は子 2 つの max。

flowchart TB
  R[root] --> L1["[0,4)"]
  R --> R1["[4,8)"]
  L1 --> L2["[0,2)"]
  L1 --> L3["[2,4)"]
  L2 --> A0[A0]
  L2 --> A1[A1]

Step 2: 1 点更新

葉を書き換え、親方向へ max を再計算。深さは log(size) なので O(log N)。

flowchart TB
  L[葉を更新] --> P[親 = max 子 2 つ]
  P --> P2[祖父 = max 子 2 つ]
  P2 --> R[root へ]

Step 3: 区間クエリ

半開区間 [l, r) を再帰的に分解:

  • 重ならない → 単位元 (max なら -∞) を返す
  • 完全に含まれる → そのノードの値を返す
  • 部分的に重なる → 左右の子に分けて max

詰まりやすい点

  • 単位元の選択を間違えるとバグる。max → -∞、sum → 0、min → +∞、GCD → 0。
  • 半開区間 [l, r) で書くと境界処理が一貫する。
  • 区間更新 + 1 点取得には 遅延セグ木。さらに区間更新 + 区間取得は遅延伝播も。
