法則 A48

焼きなまし法 — 確率で悪化も受理する

温度に応じて、悪い変更も少し受け入れる

山登り法 (A47) の欠点 (局所最適への陥り) を 「確率的に悪化を受理する」 ことで回避するヒューリスティック。

受理確率 exp((new_score - cur_score) / T) (最大化の場合)。温度 T を時間とともに下げていく ことで、序盤は広く探索 → 終盤は厳しく改善のみ受理、という挙動を作る。マラソンマッチの定番。

Step 1: 山登りと同じ近傍操作

ランダムに 2-opt 反転などを試す。これは山登りと同じ。

Step 2: 受理判定を確率に

改善なら必ず採用。悪化なら 確率 exp(Δ / T) で採用 (Δ は変化量、最大化なら正が改善)。

T が大きい (序盤) → 多少の悪化も受理 → 広く探索 T が小さい (終盤) → 悪化を拒否 → 山登りに近づく

flowchart LR
  D{Δ > 0?}
  D -- Yes --> A[必ず受理]
  D -- No --> P["確率 exp(Δ/T) で受理"]

Step 3: 温度スケジュール

時間 t に対し、T = T_start × (T_end / T_start)^(t / T_max) のように 指数的に冷却。T_start = 大きめ、T_end = 小さめ を経験的に決める。

詰まりやすい点

  • 温度のチューニングは経験頼り。テストデータで何度か試して調整。
  • exp は浮動小数のオーバーフローに注意。事前に Δ をクランプ。
  • 制限時間ギリギリまでループを回す (例: while time.time() - start < 0.95)。
