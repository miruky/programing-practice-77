法則 A39

区間スケジューリング — 終了時刻でソート + 貪欲

終了が早いものから取れば、必ず最適

N 個の区間から 重ならないように最大個数選ぶ 古典問題。「終了時刻で昇順ソートし、左から見て採用可能なら採用」という貪欲法が 必ず最適 であることが証明されています。

「他の貪欲基準 (開始時刻順、長さ順) はダメ」というのが意外な点。終了時刻が一番早いものを取れば、残った時間に他の区間を入れる余地が最大化される、という直感が証明の核心。

Step 1: 終了時刻でソート

全区間を (終了, 開始) のペアでソート。終了時刻の昇順が基準。

flowchart LR
  R[元の区間群] --> S[終了時刻昇順にソート]

Step 2: 左から貪欲に採用

現在の「最後に採用した区間の終了時刻 cur_end」を持ち、次の区間の開始 ≥ cur_end なら採用。

flowchart TB
  S["sorted intervals"] --> L[for each]
  L --> C{start >= cur_end?}
  C -- Yes --> A[採用: cur_end = end, cnt += 1]
  C -- No --> SK[スキップ]

Step 3: 正しさの証明

「最適解 OPT に対し、終了が最も早い区間 g を採用して残りを処理する貪欲も同じ個数取れる」を交換論法で示す。直感的には 早く終わる = 残り時間に余裕 が増える。

詰まりやすい点

  • 「重なる/重ならない」の境目: 端点が一致するときに重なるか? は問題による。
  • 開始時刻ソート や 長さソート にすると反例が作れて WA。
  • 「給料最大化」のような重み付きは別の DP (B39, C15) が必要。
