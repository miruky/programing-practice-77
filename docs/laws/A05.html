法則 A05

三重ループ全探索 — O(N³) でもまだ戦える

3 つの軸を独立に試したいとき

「3 つの数の組 (x, y, z) を試して、合計が K になるか?」のような問題は三重 for で書けます。計算量は O(N³)。N=100 で 10^6 なので余裕、N=500 で 1.25×10^8 で Python だと際どい、というイメージ。

三重ループまで膨らむときは、しばしば 1 つの軸を 「2 つを決めれば 3 つ目は自動で決まる」 という考えで O(N²) に落とせます。全探索でまず通してから、無駄を削っていくのが安全。

Step 1: O(N³) で間に合うかを最初に見積もる

N=100 で 10^6 (一瞬)、N=300 で 2.7×10^7 (やや遅い)、N=1000 で 10^9 (不可)。「N が小さい」と書いてあれば三重ループが想定解のことが多い。

flowchart LR
  A[N=100<br/>10^6] --> B[OK]
  C[N=300<br/>2.7x10^7] --> D[Python: 際どい]
  E[N=1000<br/>10^9] --> F[不可]

Step 2: O(N²) に落とす発想

x, y を決めた時点で z = K - x - y の 1 通りに固定できます。あとは z が有効か (集合に含まれるか) を set で O(1) チェックすれば全体 O(N²)。

flowchart LR
  X["x を選ぶ O(N)"] --> Y["y を選ぶ O(N)"]
  Y --> Z["z=K-x-y を計算"]
  Z --> S["set で存在判定 O(1)"]

Step 3: 重複の数え方に注意

(1,2,3) と (3,2,1) を区別するか? 同じ数を複数回使えるか? 問題文を最初に読んで、ループ範囲を i < j < k のように絞るかを判断する。

詰まりやすい点

  • 三重 for は深くなるほど 添字ミスが起きやすい。手元で N=3 程度で必ず手計算。
  • 「i, j, k が異なる」を要求されているなら i, j, k 全部に差をつける条件を入れる。
  • O(N³) で間に合わないとわかった時点で、O(N²) や O(N log N) に落とせないか検討する。
