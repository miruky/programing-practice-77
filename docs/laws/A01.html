法則 A01

標準入出力 — プログラムの入口と出口

まず「読む・書く」だけの最小プログラムを書けるようになる

競技プログラミングでは、入力を 標準入力 から受け取り、答えを 標準出力 に書き出します。Python では input() と print() がその窓口で、他のすべての法則はこの土台の上に乗っています。

ここを「呼び出すだけの儀式」と捉えてしまうと、後で int() の付け忘れや、改行・空白の罠で詰まります。1 行ずつ 「どの変数に、どんな型で何が入るか」 を意識して書き始めましょう。

Step 1: 入力の流れを掴む

input() は標準入力から 1 行を 文字列 として取得します。整数として使いたければ int(input()) で変換します。複数の数字が空白区切りで並んだ行は map(int, input().split()) で個別の整数列にできます。

flowchart LR
  A[標準入力<br/>'5'] -->|input| B["'5' (str)"]
  B -->|int| C[5 int]
  C --> D[計算]
  D -->|print| E[標準出力<br/>'25']

Step 2: print の挙動

print(x) は x を文字列化して書き出し、最後に改行を入れます。改行を入れたくない場合は print(x, end='') のように指定します。複数の値を空白区切りで並べる際は print(a, b, c) で十分です。

flowchart TB
  P[print x] --> S[str x]
  S --> O[stdout に書き込み]
  O --> N[改行を追加]

Step 3: 計算は中間に挟むだけ

「入力 → 計算 → 出力」の 3 段で必ず書けます。A01 は計算が N * N の 1 行だけですが、後の問題でもこの 3 段の骨格は変わりません。

詰まりやすい点

  • int() を忘れて文字列のまま掛け算しないように。'5' * '5' は型エラー。
  • input() を 2 回呼ぶと 2 行読みます。1 行に複数の値があるときは split() で 1 回の input() から分解する。
  • 末尾の改行が出力ジャッジで WA の原因になることがあります。 print() は自動で改行を付けるので安心です。
