法則 A31

包除原理 — 3 または 5 の倍数を数える型

A ∪ B = A + B − A ∩ B。重なりは 1 回だけ引く

「3 の倍数または 5 の倍数」のような 「または条件」 を数えるとき、単純に「3 の倍数 + 5 の倍数」とすると、15 の倍数を 二重カウント してしまう。

包除原理は |A ∪ B| = |A| + |B| − |A ∩ B| という公式を使い、二重に数えた分を 1 回引いて補正する考え方。集合 3 個以上は符号を交互に + / − で増やす 一般形に拡張できます。

Step 1: 2 集合の包除

|A ∪ B| = |A| + |B| − |A ∩ B|。3 の倍数の個数を A、5 の倍数の個数を B、15 の倍数の個数を A∩B とすると、「3 か 5 の倍数」は A + B − A∩B。

flowchart LR
  A[3 の倍数] --> S["|A| + |B|"]
  B[5 の倍数] --> S
  AB[15 の倍数] --> M["−|A∩B|"]
  S --> R[答え]
  M --> R

Step 2: 3 集合の包除

|A ∪ B ∪ C| = |A|+|B|+|C| − |A∩B|−|B∩C|−|A∩C| + |A∩B∩C|。奇数集合は +、偶数集合は − の符号則。

flowchart LR
  S1[1 個ずつ: +] --> R[結果]
  S2[2 個ずつ: -] --> R
  S3[3 個全部: +] --> R

Step 3: 倍数の個数

N 以下の k の倍数の個数は N // k。A∩B = lcm(3, 5) = 15 の倍数の個数 = N // 15。包除と LCM (A27) のコンボが基本。

詰まりやすい点

  • 「A∩B は AかつB」と「A∪B は AまたはB」。日本語と記号を混同しない。
  • 3 集合以上は符号則を間違えやすい。要素数 k の組には符号 (-1)^(k+1)。
  • 「ちょうど 1 つ満たす」のような問題は、包除を入れ子で適用する応用形。
