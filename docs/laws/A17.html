法則 A17

DP + 経路復元 — ゴールから戻って『どう来たか』を再構築

DP は答えを出す。経路復元はその答えをたどる

DP で最適値が求まったあと、「どの選択をして その値に辿り着いたか」を答える問題。DP テーブルは最適値だけしか持っていないので、逆方向にトレースバック して経路を組み立てます。

発想は単純: ゴールから「dp[i] = dp[i-1] + cost を満たすか?」を見て、満たせばその遷移で来たことが確定。DP の意味が固まっていれば、復元側の実装は遷移式の逆引きに過ぎません。

Step 1: 普通の DP を解く

A16 と同じく、dp[i] = min(dp[i-1]+A[i], dp[i-2]+B[i]) を完成させる。

flowchart LR
  S[スタート 1] --> M["dp[1..N] を埋める"]
  M --> G[ゴール N に最適値]

Step 2: ゴールから逆向きに辿る

現在位置 Place = N から始めて、「dp[Place-1] + A[Place] == dp[Place] なら 1 個前から来た」と判定して Place -= 1 または Place -= 2。

flowchart TB
  G[Place = N] --> C{1 個前から来たか?}
  C -- Yes --> M1[Place -= 1]
  C -- No --> M2[Place -= 2]
  M1 --> L["答えに Place を記録"]
  M2 --> L
  L --> G

Step 3: 順序を反転して出力

ゴール → スタートの順に記録したリストを reverse() してスタート → ゴール順に並べ替え、そのまま出力。

詰まりやすい点

  • 遷移が 同点 (両方の式が成り立つ) のときの分岐順で結果が変わる場合がある。 問題文の指定 (辞書順最小など) を確認。
  • 復元中、Place == 1 (スタート) に到達したら停止条件を入れる。
  • 復元方向 (ゴール→スタート) と出力方向 (スタート→ゴール) は逆。reverse 忘れに注意。
