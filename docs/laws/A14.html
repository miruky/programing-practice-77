法則 A14

半分全列挙 — N 個を半分に分け、2^(N/2) で全列挙

2^40 は無理。2^20 × 2 なら間に合う

N 個の要素から部分集合を選ぶ全探索は 2^N。N=40 だと 10^12 で完全に不可能。半分に分けて 2^(N/2) ずつ全列挙し、片方を二分探索 / set で照合する と計算量は O(2^(N/2) · N) に。

N=40 でも 2^20 ≒ 10^6 × N で間に合う。「N=30〜40 で部分集合の最適化」と書いてあったら半分全列挙の出番。

Step 1: 半分に分ける

前半 L1 = A[0:N//2]、後半 L2 = A[N//2:]。それぞれの部分集合の合計を Sum1, Sum2 として全列挙 (各 2^(N/2) 個)。

flowchart LR
  N["N 個"] --> L1[前半: N/2 個]
  N --> L2[後半: N/2 個]
  L1 --> S1[Sum1: 2^N/2 通り]
  L2 --> S2[Sum2: 2^N/2 通り]

Step 2: 一方をソート

Sum2 をソートしておくと、Sum1[i] + Sum2[j] == K を満たす Sum2[j] を二分探索で O(log) で見つけられる。

flowchart LR
  S2[Sum2 をソート] --> B["各 Sum1 i に対し<br/>K-Sum1 i を二分探索"]
  B --> ANS[該当する組が存在すれば Yes]

Step 3: 計算量

前計算 O(2^(N/2) · N)、照合 O(2^(N/2) · log)。全体 O(2^(N/2) · N)。N=40 で約 4·10^7 オーダー。

詰まりやすい点

  • 部分集合の全列挙は for mask in range(1<<n): ... でビット全探索。 mask >> j & 1 で j 番目を選ぶか判定。
  • 「ちょうど K」を探すなら set で O(1) 照合の方が簡単。
  • N が奇数のときは前半・後半のサイズが 1 違う。Sum2 の方を大きめに取ると無難。
