法則 A54

連想配列 — 辞書による key-value 管理

ハッシュで平均 O(1) アクセス

「名前 → 値」のような任意のキーで O(1) アクセスできる データ構造。Python では dict がそれ。内部はハッシュテーブルで、平均 O(1) で挿入・参照。

出現回数のカウント (collections.Counter)、グループごとの集計、メモ化など、競プロで最も多用するデータ構造の 1 つ。

Step 1: 基本操作

  • 挿入/更新: d[key] = value
  • 取得: d[key] (存在しないと KeyError)
  • 安全に取得: d.get(key, default)

全て平均 O(1)。

flowchart LR
  K[key] --> H[hash]
  H --> S[内部スロット]
  S --> V[value]

Step 2: Counter で頻度集計

from collections import Counter
c = Counter(A)
# c['x'] = 'x' の出現回数 (なければ 0)

競プロで最頻出のクラス。most_common(k) で上位 k 件を取れる。

Step 3: defaultdict で初期値自動

defaultdict(int) は新しいキーへのアクセス時に自動で 0 を返す。「キーが存在しないかもしれない」のチェックが省略でき、コードが短くなる。

詰まりやすい点

  • 同じキーを 2 回 push すると、値は上書きされる。Counter なら自動で加算してくれる。
  • ハッシュ攻撃を意識する競技は稀。素直に dict を使って OK。
  • イミュータブル (tuple, str) しかキーにできない。list はキーにできない。
