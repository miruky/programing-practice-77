法則 A04

進法変換 — 10 進数と 2 進数を行き来する

数を「位」で分解する。コンピュータ世界の母国語

10 進数の 5 は、2 進数では 101、つまり 1·4 + 0·2 + 1·1 です。進法変換は、数を 位 (= 重み) に分解して考える練習です。

ビット演算 (A23, A70 など) や繰り返し二乗法 (A29) は、「数の 2 進数表記」を直接扱います。ここで進法の扱いに慣れておくと、後のビット系の問題が一気に読みやすくなります。

Step 1: 上位桁から見る

10 桁 (= 2^9) で表すなら、まず 2^9 = 512 の桁を N // 512 % 2 で取り出し、次に 2^8 = 256 の桁、… と上位から並べると 2 進数になります。

flowchart LR
  N[N=5] --> A["N//8 %2 = 0"]
  N --> B["N//4 %2 = 1"]
  N --> C["N//2 %2 = 0"]
  N --> D["N//1 %2 = 1"]
  A & B & C & D --> R[0101]

Step 2: 下位桁から逆順に作る別解

N % 2 で最下位ビットを取り、N //= 2 で右シフト、を繰り返して逆順に積む方法もあります。短く書けますが、出力時に 逆順に並べ替える ステップが必要。

flowchart LR
  N0[N=5] --> M0[5 mod 2 = 1]
  M0 --> N1[N=2]
  N1 --> M1[2 mod 2 = 0]
  M1 --> N2[N=1]
  N2 --> M2[1 mod 2 = 1]
  M2 --> O["['1','0','1']  → 逆順 → 101"]

Step 3: 2 進数文字列 → 整数

逆方向の変換は int(s, 2) 一発。学習目的では for c in s: x = x * 2 + int(c) のように、「位を 1 つ進めるたびに 2 を掛ける」流儀で書けると進法の理解が深まります。

詰まりやすい点

  • 「0 詰めの桁数」を間違えると WA。10 桁指定なら必ず 10 桁出す。
  • Python の bin(N) は '0b101' のように接頭辞が付くので [2:] で剥がす。
  • 文字 '0' と整数 0 は別物。int(c) を入れ忘れない。
