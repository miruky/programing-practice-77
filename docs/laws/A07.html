法則 A07

いもす法 — 区間加算を端点だけで処理する

区間に +1 する代わりに、両端だけ +1 / −1 して最後に累積和

区間 [l, r] に +v を加算するクエリが Q 個あって、最終的な各位置の値を知りたい。毎回区間全体を更新すると O(Q · N) ですが、いもす法 を使うと O(Q + N)。

発想は単純: 加算開始の l に +v、加算終了の次 r+1 に -v の印だけ置く。全部置き終わってから 1 次元累積和を取れば、印が 面に広がって 元の加算を再現できます。

Step 1: 印を置く

差分配列 D[1..N+1] を 0 で初期化。各クエリ [l, r] += v についてD[l] += v、D[r+1] -= v を実行するだけ。各クエリ O(1)。

flowchart LR
  Q["クエリ +v, [l,r]"] --> A["D[l] += v"]
  Q --> B["D[r+1] -= v"]

Step 2: 累積和で「面」に戻す

全クエリを処理した後、D の累積和を取れば、各位置の最終値が一気に求まる。

flowchart LR
  D[差分配列 D] --> C["S[i] = S[i-1] + D[i]"]
  C --> R[各位置の最終値]

Step 3: 計算量

Q クエリ + N 個の累積和 = O(Q + N)。素朴な O(Q · N) と比べ、N=Q=10^5 なら 10^10 → 2·10^5 まで一気に圧縮されます。

詰まりやすい点

  • 右端の -v は r+1 に置く。r に置くと r 自身も加算が止まる ことになり 1 個ずれる。
  • 配列サイズは N+2 取ると D[r+1] で IndexError を起こさない。
  • 区間が半開 [l, r) で来たら、-v は r に置く。問題文の区間記法をまず確認。
