法則 A23

bitDP — 集合を整数 1 個に圧縮

2^N 通りの集合を 0..2^N-1 の整数で表す

「どの要素を選んだか」の集合 S を、N ビットの整数 として扱う DP。状態数が 2^N (= N=20 で約 10^6) なので、N が小さい全探索の代用品 として使います。

TSP (巡回セールスマン) を O(2^N · N²) で解いたり (B23)、クーポン券問題 (A23) を解いたりするのが代表例。ビット演算 (AND, OR, XOR, シフト) に慣れるのが第一歩。

Step 1: ビットと集合の対応

整数 state の k ビット目が 1 ⇔ 要素 k が集合に含まれる。1 << k で「k 番目だけが ON」の集合を作れる。

flowchart LR
  S["state = 5 (=101 in binary)"] --> A[bit0 = 1: 要素0 含む]
  S --> B[bit1 = 0: 要素1 含まない]
  S --> C[bit2 = 1: 要素2 含む]

Step 2: 集合演算

  • 追加: state | (1 << k)
  • 削除: state & ~(1 << k)
  • 反転: state ^ (1 << k)
  • 含む?: (state >> k) & 1
  • 和集合: s1 | s2
  • 共通: s1 & s2

Step 3: DP の形

dp[i][state] = i 枚目まで処理し、状態 state を作る最小コスト のような 2 次元 DP。クーポン問題なら state は「無料になっている品物の集合」。

遷移: 「i 枚目を使わない → state そのまま」「i 枚目を使う → state | coupon_i に行く」。

flowchart LR
  P["dp[i-1][state]"] --> N1["dp[i][state] (使わない)"]
  P --> N2["dp[i][state | coupon_i] + 1 (使う)"]

詰まりやすい点

  • 状態数は 2^N。N=20 が現実的な上限 (約 10^6)。N=25 までなら工夫次第。
  • ループ順は for state in range(1 << N) で 0 から大きくしていくのが典型。
  • state | next_state で集合を結合できる。これが bitDP の万能ツール。
