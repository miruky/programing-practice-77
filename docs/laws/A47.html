法則 A47

山登り法 — 改善する変更だけ受理

ランダムに動かして、良くなったら採用

ランダムに小さい変更を加え、スコアが改善した場合のみ採用 する局所探索。「2-opt 反転」のような近傍操作と組み合わせて使う。

実装は短いが、局所最適から抜け出せない弱点 がある。焼きなまし (A48) はこの弱点を確率で「あえて悪化を受理」して回避する。

Step 1: 初期解と現在スコア

貪欲 (A46) で初期解を作るか、ランダム順列から始める。現在の経路長を current_score として持つ。

Step 2: 近傍操作 (2-opt)

ランダムな区間 [l, r] を選んで反転する。TSP では「2 本の辺を 2 本の別の辺に張り替える」操作に対応。

flowchart LR
  P1["P = ...A,B,...,C,D,..."] -->|区間 B..C 反転| P2["P' = ...A,C,...,B,D,..."]

Step 3: 改善判定

反転後のスコアを計算。改善なら採用、悪化なら反転を取り消す。

if new_score < current_score:
    current_score = new_score  # 採用
else:
    P[l:r+1] = reversed(P[l:r+1])  # 取り消し

詰まりやすい点

  • 反転スコア計算は 2 辺の差分だけで O(1) に高速化できる (TSP の差分計算)。
  • 局所最適に必ず陥る。脱出には焼きなまし (A48) や乱数再起動 (Restart)。
  • 乱数の seed を固定するとデバッグが楽。
