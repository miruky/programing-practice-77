法則 A20

LCS — 最長共通部分列

2 つの文字列の「共通骨格」を測る古典 DP

2 つの文字列 S, T に共通して現れる部分列のうち、最長のものの長さ を求める。部分列は連続でなくてもよい (順序だけ保つ)。

dp[i][j] = S[0..i) と T[0..j) の LCS 長 の 2 次元 DP。「最後の文字が一致するか?」で遷移が 2 分岐するパターンは、編集距離 (B20) や 回文部分列 (B21) にも共通する万能の型。

Step 1: 状態の定義

dp[i][j] = S の先頭 i 文字と T の先頭 j 文字の LCS の長さ。初期状態は dp[0][j] = dp[i][0] = 0 (空文字列との LCS は 0)。

Step 2: 一致する場合の遷移

S[i-1] == T[j-1] なら、この文字を LCS に追加できる。dp[i][j] = dp[i-1][j-1] + 1。

flowchart LR
  P["dp[i-1][j-1]"] --> A["+1"]
  A --> D["dp[i][j]"]

Step 3: 一致しない場合

S か T のどちらかを 1 文字捨てた状態の max を取る。dp[i][j] = max(dp[i-1][j], dp[i][j-1])。

flowchart LR
  P1["dp[i-1][j] (S の最後を捨てる)"] --> M[max]
  P2["dp[i][j-1] (T の最後を捨てる)"] --> M
  M --> D["dp[i][j]"]

詰まりやすい点

  • 「最長共通部分列」と「最長共通部分文字列 (連続)」は別物。 部分文字列だと遷移が違う。
  • 0 行目・0 列目を 0 で初期化するだけで境界処理が消える。番兵の威力。
  • LCS そのもの (文字列) を復元するには A17 と同じ要領で逆向きに辿る。
