法則 A62

DFS — 深さ優先探索で連結判定

行けるだけ進んで、戻ってきたら別の道

頂点 v から 「行けるだけ進む」 探索。再帰またはスタックで実装でき、計算量は O(V+E)。

用途: 連結判定、橋・関節点検出、サイクル検出、トポロジカルソート、木 DP の前処理 など。再帰版は短く書けるが、Python は再帰上限 1000 がデフォなので大きいグラフでは要拡張。

Step 1: 再帰版

sys.setrecursionlimit(10**6)
def dfs(v):
    visited[v] = True
    for u in G[v]:
        if not visited[u]:
            dfs(u)
flowchart TB
  V[頂点 v] --> M[visited v = True]
  M --> L[全隣接 u を確認]
  L --> R{visited u?}
  R -- No --> D[dfs u 再帰]
  R -- Yes --> L

Step 2: スタック版 (非再帰)

再帰上限を気にせずに済む。深いグラフでは必須。

stack = [start]
visited[start] = True
while stack:
    v = stack.pop()
    for u in G[v]:
        if not visited[u]:
            visited[u] = True
            stack.append(u)

Step 3: 連結判定

DFS 後、all(visited[1..N]) なら連結。

用途を広げると、連結成分の個数 = DFS を始められた回数。

詰まりやすい点

  • Python の再帰上限 1000。sys.setrecursionlimit(10**6) で拡張。
  • グリッド系は (i, j) を 1 頂点として展開。隣接は上下左右 4 方向。
  • visited 配列のサイズは V+1 (1-indexed の場合)。
