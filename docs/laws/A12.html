法則 A12

答えで二分探索 — 「答えが x 以下か」を直接判定

探すのは値じゃない。OK/NG の境目

「最大値を最小化したい」「最小値を最大化したい」「ちょうど K にしたい」といった最適化問題は、「答えが x 以下か?」という判定問題 に書き換えると二分探索で解けます。これを 「答えで二分探索」(めぐる式二分探索) と呼びます。

肝は 単調性 : x を大きくすると判定結果が NG → OK と 一方向にだけ変わることが保証されている必要があります。判定関数 check(x) を 1 つ書けばよく、全探索や微妙な貪欲よりも安全。

Step 1: 単調性を確認

「時間 x 以上あれば終わるか」「容量 x 以下に収まるか」のように、x が大きくなるほど判定が易しくなる (NG → OK と切り替わる) ことを確認。

flowchart LR
  X1[x=1<br/>NG] --> X2[x=2<br/>NG]
  X2 --> X3[x=3<br/>OK]
  X3 --> X4[x=4<br/>OK]
  X4 --> X5[x=5<br/>OK]

Step 2: check(x) を実装

「x 分で K 枚印刷できる?」など、「x が与えられたら Yes/No を返す関数」を 1 つ書く。通常の解法 (貪欲、シミュレーションなど) より単純になることが多い。

Step 3: 二分探索で境界を絞り込む

check(x) が NG → OK に切り替わる 最小の x を二分探索で見つける。

flowchart TB
  S["left=1, right=10^9"] --> L[while left < right]
  L --> M["mid = (l+r)//2"]
  M --> C{check mid}
  C -- NG --> NL["left = mid + 1"]
  C -- OK --> NR["right = mid"]
  NL --> L
  NR --> L
  L --> A["left が答え"]

詰まりやすい点

  • 単調性がないと二分探索は使えない。「x を大きくしたのに NG に戻る」なら別の方法へ。
  • 整数二分探索なら while left < right、実数二分探索なら for _ in range(200)。
  • 「最大化」のときは while left < right: mid = (l+r+1)//2; ... と切り上げ版を使う (A77 で使うテンプレ)。
