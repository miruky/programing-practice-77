法則 A63

BFS — 幅優先探索で最短歩数

近い順に層を広げる。重みなし最短路の基本

頂点 s から 距離が近い順 に層状に広げる探索。キュー (deque) を使い、計算量 O(V+E)。

重みなしグラフ (またはすべての辺の重みが等しいグラフ) の最短路 を求めるのに使う。迷路、迷宮、グリッド最短路の定番。重み付きが必要ならダイクストラ (A64) に切り替え。

Step 1: 初期化

dist = [-1] * (N+1)、dist[s] = 0、q = deque([s])。

flowchart LR
  S[start s] --> D["dist[s] = 0"]
  D --> Q["q = [s]"]

Step 2: 層を広げる

キューの先頭を取り出し、隣接頂点の dist をまだ未訪問なら +1 して push。

flowchart TB
  Q[キュー先頭 v] --> P[popleft]
  P --> L[各隣接 u]
  L --> C{dist u == -1?}
  C -- Yes --> S["dist[u] = dist[v]+1; append u"]

Step 3: グリッド BFS

迷路では (i, j) を 1 頂点、隣接は上下左右 4 方向。dx, dy = [-1,0,1,0], [0,-1,0,1] の定数を使うと書きやすい。

詰まりやすい点

  • 必ず deque.popleft()。list.pop(0) は O(N) で TLE。
  • visited ではなく dist == -1 で未訪問判定すると最短距離も同時に取れる。
  • 重み付きには使えない。それはダイクストラ (A64)。
