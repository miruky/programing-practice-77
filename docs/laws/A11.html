法則 A11

二分探索 — ソート済み列の真ん中を狙う

毎回半分捨てれば、N=10^18 でも 60 回で答えに到達

ソート済みの配列で「目的の値の位置」を探すとき、毎回真ん中と比較して 半分を捨てる だけで、O(log N) で到達できます。

N=10^6 でも 20 回、N=10^18 でも 60 回程度の比較で済む脅威的な速さ。Python では bisect.bisect_left を使えば 1 行ですが、「NG と OK の境目を探す」 という考え方は次の A12 (答えで二分探索) で必須になります。

Step 1: ループ不変条件

left, right は 答えがある可能性のある範囲 を表す。毎回 mid = (left + right) // 2 で半分に分け、片側を捨てる。

flowchart LR
  L["left"] --> M["mid = (l+r)//2"]
  M --> R["right"]
  M --> J{"A[mid] と target を比較"}
  J -- target が左 --> NL["right = mid"]
  J -- target が右 --> NR["left = mid + 1"]

Step 2: bisect_left と bisect_right の違い

bisect_left(A, x) は「x を入れてもソートが崩れない最も左の位置」、bisect_right(A, x) は「最も右の位置」。重複がある場合、左境界と右境界が変わるので使い分けが必要。

flowchart LR
  A["配列: 1 2 3 3 3 5"] --> L[bisect_left x=3 = 2]
  A --> R[bisect_right x=3 = 5]
  L --> D["差 = 3 の個数"]
  R --> D

Step 3: 自前実装テンプレ

ライブラリに頼らない実装パターン:

left, right = 0, N
while left < right:
    mid = (left + right) // 2
    if A[mid] >= x:
        right = mid
    else:
        left = mid + 1
# left は『A[i] >= x となる最初の位置』

詰まりやすい点

  • 配列がソートされていない と二分探索は機能しない。最初に sort() を忘れない。
  • mid = (l + r) // 2 で l == r-1 のときに mid == l となり、 l = mid + 1 の代わりに l = mid を書いていると 無限ループ。
  • bisect_right - bisect_left で出現回数を数えるのは超頻出イディオム。
