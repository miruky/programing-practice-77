法則 A70

状態 BFS — 2^N 通りの状態を頂点に持つ

集合を整数で表し、状態空間グラフを BFS

「ランプの ON/OFF」「マスの色」のような N ビットの状態 を、整数 0..2^N-1 で表現。状態を頂点、1 回の操作を辺とするグラフ を構築し、BFS で最短手数を求める。

N = 20 でも状態数 10^6 程度で現実的。bitDP (A23) と発想は同じ で、両者は親戚のような関係。

Step 1: 状態を整数化

ランプ N 個の状態は state = Σ (lamp[i] << i) のような N ビット整数。状態数は 2^N。

flowchart LR
  L["ランプ [1, 0, 1, 1]"] --> S["state = 13 = 1011_2"]

Step 2: 状態遷移グラフを構築

各状態から、各操作で行ける次状態を計算。「ランプ x, y, z を反転」なら XOR で next = state ^ (1<<x) ^ (1<<y) ^ (1<<z)。

flowchart LR
  S[state] --> O1[操作 1: XOR]
  O1 --> N1[次状態 1]
  S --> O2[操作 2: XOR]
  O2 --> N2[次状態 2]

Step 3: BFS で最短

初期状態 (= 入力されたランプ配置) からゴール (= 全 1) までの BFS。距離が答え。

詰まりやすい点

  • 状態数 2^N がメモリに乗るか確認。N = 25 でも 33M で厳しい。
  • XOR で集合反転は強力。s ^ mask で複数 bit を同時反転できる (A70_extra)。
  • 「最短」が問題なら BFS、「到達可能性」だけなら DFS でも可。
