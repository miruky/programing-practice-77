法則 A33

Nim ゲーム — XOR が 0 でなければ先手必勝

複数山の Nim は XOR 和で勝敗が決まる

石の山が複数あり、各手番で 1 つの山から好きな数取れる ゲーム (= Nim)。取れなくなった方が負け。

驚くべき定理: 全山の石の数の XOR が 0 のとき、後手必勝。1 山だけ取り出して試行錯誤するだけでは見えない、強力な数学的構造です。Grundy 数 (A34) はこれを「Nim でない一般ゲームを Nim に変換する」一般形。

Step 1: XOR の不思議な性質

XOR が 0 なら全部 0 か、ペアがある (2 進数の各 bit で偶数個 1 が立っている)。1 つの数を変えると XOR が 0 ではなくなる。逆に、XOR が 0 でないとき、必ず「XOR を 0 にする手」が存在する。

Step 2: XOR=0 なら後手必勝

  • XOR=0 の状態: どの山から何個取っても XOR は 0 でなくなる → 相手に「XOR≠0」を渡す
  • XOR≠0 の状態: 必ず「XOR を 0 にする手」がある → 相手に「XOR=0」を渡せる
flowchart LR
  X0[XOR = 0] -->|何取っても| X1[XOR ≠ 0]
  X1 -->|0 に戻す手あり| X0

Step 3: 実装

全山の石数を XOR で集約するだけ。

xor = 0
for a in A:
    xor ^= a
print('First' if xor != 0 else 'Second')

詰まりやすい点

  • 「1 手で 最低 1 個 取る必要があるか」「0 個も OK か」でルールが変わる。標準は 最低 1 個。
  • ミゼール (最後に取った方が負け) は通常の Nim と勝敗が一部異なる。
  • 「2 次元 Nim」のような変種は、各座標の差を Nim 和に持ち込む発想 (B33)。
