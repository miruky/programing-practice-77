法則 A15

座標圧縮 — 値を「順位」に置き換える

値の絶対サイズに依存しない処理に書き換える定石

値が 10^9 まで取りうるが、要素数は 10^5 しかないとき、配列のサイズが値に依存して大きくなりがちな問題 (BIT、セグ木、頻度配列…) を「値 → 順位」 に置き換えて N サイズに圧縮するテクニック。

ソートして重複を除けば順位が決まり、bisect_left(sorted_unique, x) で順位を引ける。転倒数 (B59) や 座標による分類 の前処理として頻出。

Step 1: 重複を除いてソート

B = sorted(set(A)) で、A の値を昇順かつ重複なしの順位対応表に変換。

flowchart LR
  A["A = [50, 10, 30, 30, 80]"] --> B["B = sorted set A = [10,30,50,80]"]

Step 2: 順位に変換

各 A[i] を bisect_left(B, A[i]) で順位に変換。値 10 → 0, 30 → 1, 50 → 2, 80 → 3。

flowchart LR
  A["A = [50, 10, 30, 30, 80]"] --> R["順位 = [2, 0, 1, 1, 3]"]

Step 3: 後続処理は順位で

頻度配列の長さが len(B) で済む。BIT、セグメント木、累積和などが「値域に対して」ではなく「要素数に対して」のサイズで作れる。

詰まりやすい点

  • 重複の扱い: set は重複を除く。重複を残したい場合は sorted(A) を使う。
  • 順位は 0-indexed か 1-indexed か。BIT 系は 1-indexed が多い。
  • 元の値が必要な場合は順位 → 値のテーブル B を残しておく。
