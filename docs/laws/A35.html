法則 A35

交互最善 DP — min と max を交互に切り替える

先手は max、後手は min。階層で交互に

「2 人が交互に最善手を打つ」ゲームで、先手は自分の得を最大化、後手は先手の得を最小化するときの最終スコアを求める DP。

奇数階層では max、偶数階層では min を取るだけ。後退解析 (A32) の応用形ですが、勝敗ではなく スコア を扱うので、min/max を交互に使う点が特徴。

Step 1: 状態の定義

dp[i][j] = ピラミッドの (i, j) からゲームを開始したときの最終スコア とする。「先手の累計スコア」と読み替えてもよい。

Step 2: 最下段は値そのもの

ピラミッドの最下段 (i = N) は、その値がそのまま最終スコア。dp[N][j] = A[j-1]。

Step 3: 上段は子の max / min

奇数段は先手 = max、偶数段は後手 = min。

if i % 2 == 1:  # 先手の手番
    dp[i][j] = max(dp[i+1][j], dp[i+1][j+1])
else:           # 後手の手番
    dp[i][j] = min(dp[i+1][j], dp[i+1][j+1])
flowchart TB
  P[親 i j] --> C1[子 i+1 j]
  P --> C2[子 i+1 j+1]
  P -.奇数段 = max.- C1
  P -.偶数段 = min.- C2

詰まりやすい点

  • 「先手」「後手」のどちらが max / min を取るかは問題文を慎重に読む。
  • 階層の偶奇でなく 手番数の偶奇 で切り替える問題もある。
  • 「先手のスコア − 後手のスコア」を最大化する問題は別の DP に書き換える (= 差分 DP)。
