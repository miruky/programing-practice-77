法則 A08

2 次元累積和 — 矩形の和を O(1) で答える

左上から (i, j) までの合計を持っておけば、長方形は引き算 3 回で求まる

1 次元累積和 (A06) の 2 次元版です。Z[i][j] = 「左上 (1,1) から (i,j) までの合計」 を前計算しておけば、任意の矩形 (A,B)-(C,D) の和は Z[C][D] - Z[A-1][D] - Z[C][B-1] + Z[A-1][B-1] で O(1)。

重要なのは「最後の +Z[A-1][B-1]」。これは 包除原理 で、2 回引かれてしまった左上の長方形を 1 回戻すための補正です。

Step 1: Z[i][j] の構築

0 行目・0 列目は 0 (番兵) のまま残し、Z[i][j] = X[i-1][j-1] + Z[i-1][j] + Z[i][j-1] - Z[i-1][j-1] で埋める。「今のマス + 上 + 左 − 左上の重複」の式が、まさに小さい矩形の包除そのもの。

flowchart TB
  TL[左上 Z i-1 j-1] --> SUB["−"]
  T[上 Z i-1 j] --> ADD["+"]
  L[左 Z i j-1] --> ADD
  X[X i-1 j-1] --> ADD
  ADD --> R["Z[i][j]"]
  SUB --> R

Step 2: 矩形の和

矩形 (A,B)-(C,D) の和は次の図のように 4 つの長方形の組み合わせで表せます。「大きい長方形 − 上の余り − 左の余り + 左上の戻し」。

flowchart LR
  Z_CD["Z[C][D]"] --> R["答え"]
  Z_AD["−Z[A-1][D]"] --> R
  Z_CB["−Z[C][B-1]"] --> R
  Z_AB["+Z[A-1][B-1]"] --> R

Step 3: 2 パスで作る別実装

「横方向に累積 → 縦方向に累積」の 2 回パスでも同じ Z が得られます。包除の式を覚えるのが苦手なら、こちらの方が直感的に書ける場面もあります。

flowchart LR
  X[元配列 X] -->|横累積| H[各行が前から累積された配列]
  H -->|縦累積| Z[2 次元累積和 Z]

詰まりやすい点

  • 配列の境界 (0 行目・0 列目) を 必ず 0 で残す。番兵がないとクエリ式の -1 がマイナス添字になる。
  • 包除の +Z[A-1][B-1] を忘れがち。1 つでも符号を間違えると致命的。
  • 半開区間と閉区間の混在に注意。問題文の (A,B)-(C,D) が閉なら今回の式、半開なら -1 をすべて取り除く。
