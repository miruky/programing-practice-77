法則 A13

しゃくとり法 — 左右の窓を一方向に動かす

右端は戻らない。広げては縮めるだけで O(N)

「条件を満たす連続区間の個数」「最長の連続区間」のような問題で、左端と右端の 2 つのポインタを一方向にだけ進める ことで全体 O(N) を達成する技法。

二重 for で書くと O(N²) になるところを、「左端を 1 つ進めるたびに、右端は前回の続きから動く」という性質を利用して右端を 巻き戻さないことで、全体の右端の動きも合計 N 回に抑えます。

Step 1: 単調性が必要

「左端 l のとき、条件を満たす最大の右端は R(l)」が l について単調 であること。つまり l を進めると R(l) も進む (戻らない)。これがしゃくとり法の前提。

flowchart LR
  L1[l=1] --> R1[R=4]
  L2[l=2] --> R2[R=5]
  L3[l=3] --> R3[R=5]
  L4[l=4] --> R4[R=7]

Step 2: 窓を広げる

右端 r を「条件を満たす限り」進める while ループを内側に書く。条件を破った時点で停止し、その時の r-l などを答えに加算。

flowchart LR
  L[l を固定] --> W[while 条件 OK と r 範囲内]
  W -->|r 進める| W
  W -->|止まったら| C[r-l などを集計]

Step 3: 左端を進める

次のターンで l += 1。r は戻さない。これが O(N) を達成する核心。

flowchart LR
  S1[l=1, r=1..4] --> S2[l=2, r=4..5]
  S2 --> S3[l=3, r=5..5]
  S3 --> S4[l=4, r=5..7]

詰まりやすい点

  • 「条件を満たす最大の r」と「条件を破る最小の r」を取り違えると個数が 1 ずれる。
  • r を毎回 l に戻すと O(N²) になり、しゃくとりの意味がなくなる。
  • 区間和 / 区間 GCD / 区間 OR など、計算量 O(1) で更新できる量に向く。
