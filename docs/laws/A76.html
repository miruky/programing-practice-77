法則 A76

区間 DP の高速化 — 累積和で遷移を O(1) に

DP 内の `Σ` を累積和で潰す典型

区間 DP の遷移に 「区間内の和」 が出てくるとき、累積和 (A06) を一緒に持っておくと 1 つの遷移が O(1) になる。

全体 O(N²) の DP の中の各遷移が O(N) なら全体 O(N³)、O(1) に潰せば O(N²)。「DP + 累積和」「DP + セグメント木」など、DP の遷移を別データ構造で加速 するパターンは応用範囲が広い。

Step 1: DP 遷移内の Σ を見つける

dp[i] = max_k (dp[k] + Σ_{j=k}^{i} ...) のような形で、和 Σ が遷移内に毎回出てくるなら 累積和の出番。

flowchart LR
  D["dp[i] の遷移"] --> S["Σ が含まれる?"]
  S -- Yes --> A[累積和で O 1 化]

Step 2: 累積和を前計算

Σ の対象配列について累積和 S[i] を計算。Σ_{j=l}^{r} A[j] = S[r] - S[l-1] で O(1) に。

Step 3: DP に組み込む

DP の遷移を累積和を使って書き直す。二重ループの全体は変わらないが、内側の Σ が消えて O(N²) に。

flowchart LR
  D["DP O(N²)"] --> T[各遷移 O 1]
  T --> R[全体 O N²]

詰まりやすい点

  • 累積和の範囲を間違えると遷移がズレる。手で 1 ステップ確認 する。
  • 累積和ではなく 累積 max / min が必要なケースも (セグ木が必要になる)。
  • DP 自体の状態数を減らせないか? も一緒に検討。
