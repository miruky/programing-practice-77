77 法則ライブラリ

鉄則本に登場する 77 個の法則を、初心者向けの図解 (mermaid) と解説でまとめました。 本を読んでいなくても、各法則の使いどころ・やり方・落とし穴がここだけで分かるように作っています。

1章

アルゴリズムと計算量

計算量・全探索の基礎を学ぶ章。アルゴリズムの第一歩。
A01

標準入出力 — プログラムの入口と出口

まず「読む・書く」だけの最小プログラムを書けるようになる

A02

線形探索 — 端から順に全部見る

まずは全部見る。それで間に合うなら勝ち

A03

二重ループ全探索 — 2 つの軸を独立に試す

2 重の for 文だけで多くの問題が殴れる

A04

進法変換 — 10 進数と 2 進数を行き来する

数を「位」で分解する。コンピュータ世界の母国語

A05

三重ループ全探索 — O(N³) でもまだ戦える

3 つの軸を独立に試したいとき
2章

累積和

区間クエリを高速化する累積和テクニック。1次元・2次元・いもす法まで。
A06

1 次元累積和 — 区間和を O(1) で答える

先に「先頭からの合計」を作っておけば、引き算で区間和が出る

A07

いもす法 — 区間加算を端点だけで処理する

区間に +1 する代わりに、両端だけ +1 / −1 して最後に累積和

A08

2 次元累積和 — 矩形の和を O(1) で答える

左上から (i, j) までの合計を持っておけば、長方形は引き算 3 回で求まる

A09

2 次元いもす法 — 矩形加算を 4 隅で表現

長方形を毎回塗らない。四隅に + / − を置いて累積和で面に戻す

A10

区間最大値クエリ — 前後 2 方向の累積最大値

累積和の「max 版」。両端から max を取れば真ん中の答えが出る
3章

二分探索

ソート済み列の探索と「答えで二分探索」の発想を獲得する章。
A11

二分探索 — ソート済み列の真ん中を狙う

毎回半分捨てれば、N=10^18 でも 60 回で答えに到達

A12

答えで二分探索 — 「答えが x 以下か」を直接判定

探すのは値じゃない。OK/NG の境目

A13

しゃくとり法 — 左右の窓を一方向に動かす

右端は戻らない。広げては縮めるだけで O(N)

A14

半分全列挙 — N 個を半分に分け、2^(N/2) で全列挙

2^40 は無理。2^20 × 2 なら間に合う

A15

座標圧縮 — 値を「順位」に置き換える

値の絶対サイズに依存しない処理に書き換える定石
4章

動的計画法

DP の典型パターン（部分和・ナップサック・LCS・LIS など）を網羅。
A16

1 次元 DP の基本 — dp[i] の意味を 1 行で言える

状態 → 遷移 → 答えの 3 段で書く

A17

DP + 経路復元 — ゴールから戻って『どう来たか』を再構築

DP は答えを出す。経路復元はその答えをたどる

A18

部分和問題 — 「i 枚から合計 j 作れるか」の bool DP

数値を作れるかを True/False で広げていく

A19

ナップサック問題 — 重さと価値の 2 次元 DP

DP の王様。重量制限のもとで価値を最大化

A20

LCS — 最長共通部分列

2 つの文字列の「共通骨格」を測る古典 DP

A21

区間 DP — 区間 [l, r] を状態にする

小さい区間から大きい区間へ。LEN を外ループに置くのが定石

A22

すごろく型 DP — 前向き遷移で書く

自分の dp が決まったら、次の dp を更新する

A23

bitDP — 集合を整数 1 個に圧縮

2^N 通りの集合を 0..2^N-1 の整数で表す

A24

LIS — 最長増加部分列

O(N log N) で部分列の最長を求める伝説のアルゴリズム

A25

経路数え上げ — 2 次元 DP で数える典型

格子上の経路数。障害物を `0` で消すだけ
5章

数学的問題

素数判定・GCD・モジュラ逆元・ゲーム理論（Grundy 数）まで。
A26

素数判定 — √N までの試し割り

N の約数は必ず √N までに 1 つは入っている

A27

ユークリッド互除法 — 最大公約数を O(log) で

`gcd(a, b) = gcd(b, a mod b)` を繰り返すだけ

A28

mod 計算 — 毎回余りを取って小さく保つ

加減乗は mod の中でも成り立つ。除算だけは別扱い

A29

繰り返し二乗法 — a^b を O(log b) で

b を 2 進法で分解して、必要なベキだけ掛ける

A30

mod 逆元 — フェルマーの小定理で割り算を作る

素数 M なら b⁻¹ ≡ b^(M-2) (mod M)

A31

包除原理 — 3 または 5 の倍数を数える型

A ∪ B = A + B − A ∩ B。重なりは 1 回だけ引く

A32

ゲーム後退解析 — 終端から勝ち負けを伝播

勝てる状態 = 1 手で『相手が負ける状態』へ行ける

A33

Nim ゲーム — XOR が 0 でなければ先手必勝

複数山の Nim は XOR 和で勝敗が決まる

A34

Grundy 数 — 独立ゲームを Nim に変換する

各部分ゲームの Grundy 数の XOR が 0 なら後手必勝

A35

交互最善 DP — min と max を交互に切り替える

先手は max、後手は min。階層で交互に
6章

考察テクニック

全探索・貪欲・偶奇・上界下界など、思考力を鍛える章。
A36

偶奇の不変量 — Parity は変わらない

操作で「偶奇」が変わらないなら、その偶奇を見れば必ず答えが分かる

A37

貢献度分解 — 各要素を独立に数える

「全部の和」を「各要素が答えに何回登場するか」で分解

A38

左右からの累積 — 各位置の上限を独立に求める

左から見た条件と右から見た条件、両方を満たす最大値

A39

区間スケジューリング — 終了時刻でソート + 貪欲

終了が早いものから取れば、必ず最適

A40

個数集計と組合せ数え上げ — 値ごとに数えて C(n, k) で合計

同じ値の組を数えるなら、値ごとに分けて組合せを足す

A41

鳩ノ巣原理 — 「必ず同じ箱に 2 つ入る」

N+1 個を N 個の箱に入れたら、必ずどこかに 2 個入る

A42

2 軸を全探索 — (a, b) の組を全部試す

100 × 100 = 10^4 通りなら、毎回判定しても余裕

A43

上界の最大 — 「各物体の最短距離」の max

「全体最小」を「個別最大」に分解

A44

遅延処理 — 物理操作の代わりにフラグ管理

「実際に動かす」のはコストが大きい。代わりに方向や順序だけを記録

A45

不変量の発見 — 操作で保たれる量を見抜く

A36 の発展形。総和、mod、bit ごとの偶奇など
7章

A48

A49

A50

データ構造とクエリ処理

スタック・キュー・ヒープ・ハッシュ・ダブリング・セグメント木まで。

A52

A54
