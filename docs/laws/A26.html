法則 A26

素数判定 — √N までの試し割り

N の約数は必ず √N までに 1 つは入っている

N が素数かを判定するには、2 から √N までの整数で割れるか を試すだけ。計算量 O(√N)、N=10^12 でも 10^6 回程度の試し割りで判定できます。

鍵は 「N の約数の組 (a, b) (a≤b) は a ≤ √N」 という性質。片方を √N まで探せば、もう片方は自動で見つかる。約数列挙 (C04) やエラトステネス (B26) もこの性質の応用です。

Step 1: なぜ √N まででよいか

N = a × b (a, b は整数, a ≤ b) なら、a² ≤ N、つまり a ≤ √N。なので a を √N まで探せば、ペアの b は自動で対応する数になる。

flowchart LR
  N["N = a × b"] --> C["a ≤ b"]
  C --> R["a² ≤ N"]
  R --> S["a ≤ √N"]

Step 2: 試し割り

for i in range(2, int(N**0.5) + 1): if N % i == 0: 素数でない。途中で 1 つでも割り切れたら合成数。

flowchart TB
  L["i = 2, 3, ..., √N"] --> C{N mod i == 0?}
  C -- Yes --> NO[合成数]
  C -- No --> L
  L --> P[素数]

Step 3: 例外処理

N=1 は素数ではない。if N < 2: return False を入れておく。int(N0.5) は浮動小数の誤差で 1 ずれることがある。int(N0.5) + 1 で安全側に倒す。

詰まりやすい点

  • N=1 を素数と判定しないように。
  • 浮動小数の N**0.5 は誤差を含む。isqrt(N) (Python 3.8+) を使うと厳密。
  • 「N 以下の素数を全部欲しい」なら エラトステネスのふるい (B26) に切り替え。
