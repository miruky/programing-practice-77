法則 A46

貪欲法 (Nearest Neighbor) — その場で最良を取る

ヒューリスティック問題の最初の一歩

TSP のような「巡回経路を求めて最短に」というヒューリスティック問題で、現在地から最も近い未訪問都市へ移動 する単純な貪欲。

最適解は保証しないが、実装が単純で初期解として優秀。山登り (A47) や焼きなまし (A48) の出発点として使う。競プロのマラソンマッチでも「まず貪欲」が定石。

Step 1: 初期都市を決める

都市 0 から出発と決める。visited[0] = True、経路 P = [0]。

flowchart LR
  S[都市 0] --> N1[次に行ける都市 1..N-1]

Step 2: 最近傍を選ぶ

未訪問の中で 現在地から最も近い都市 を全探索 O(N) で見つける。選んだら経路に追加、訪問済みにする。

flowchart TB
  C[現在地] --> S[未訪問都市を全部見る]
  S --> M[最小距離を探す]
  M --> A["P.append(min_id)"]
  A --> V["visited[min_id] = True"]

Step 3: 最後にスタートへ戻る

N 回繰り返したあと、P.append(0) でループを閉じる。経路長は Σ dist(P[i], P[i+1])。

詰まりやすい点

  • 貪欲は 最適性なし。「だいたい良い」答えしか出ない。
  • 同距離の都市が複数あるときの選び方で結果がぶれる。
  • 局所最適に陥りやすい。改善には山登り (A47) や焼きなまし (A48)。
