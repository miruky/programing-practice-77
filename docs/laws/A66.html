法則 A66

Union-Find — 連結成分の素集合データ構造

2 つの要素が同じグループか? 結合する。それだけ

「同じグループか?」「グループを併合する」 の 2 操作を ほぼ O(1) で行うデータ構造。経路圧縮 と Union by size/rank を組み合わせれば、1 操作あたり O(α(N)) ≒ O(1) という驚異の速さ。

MST (A67)、オフライン連結性 (B66)、グラフの連結成分など、多くの問題で土台となる。「辺を追加していくときの連結性」が得意。辺の削除は苦手 (時系列を逆順にして追加に変換するのが定石)。

Step 1: 各要素の親を持つ

par[x] = x の親 (= 同じグループの代表)。初期状態は par[x] = -1 (各自が根)。

flowchart LR
  P1[par 1 = -1] -.- G1[1 単独]
  P2[par 2 = -1] -.- G2[2 単独]

Step 2: root(x) — 根を辿る

par[x] != -1 の間、親に進む。経路圧縮: 辿った頂点を全部 根に直結 して、次回以降高速化。

flowchart TB
  X[x] --> P[par x]
  P --> P2[par par x]
  P2 --> R[根]
  X -.経路圧縮で直接.-> R

Step 3: unite(u, v) — 結合

root(u) と root(v) が違うなら、小さい木を大きい木に繋ぐ (union by size)。サイズが揃うと木が深くならない。

詰まりやすい点

  • 経路圧縮なしの素朴版は 最悪 O(N)。両方の最適化を必ず入れる。
  • 辺削除には弱い。オフライン + 時系列逆順 + 追加 に書き換えるのが王道。
  • 配列サイズ N+1 にして 1-indexed に揃えると問題と一致して読みやすい。
