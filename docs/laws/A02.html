法則 A02

線形探索 — 端から順に全部見る

まずは全部見る。それで間に合うなら勝ち

配列の中に目的の値があるかを知りたいとき、最も素朴な方法が 線形探索 です。「先頭から最後まで 1 個ずつ照合する」だけのアルゴリズムで、計算量は O(N)。

二分探索やハッシュよりも遅いと思われがちですが、N がせいぜい 10^6 程度なら線形探索で十分間に合うことが多く、余計なバグの心配もありません。「速さよりまず正しさ」を学ぶ第一歩です。

Step 1: 全部見る = O(N)

配列の要素数 N 個を 1 回ずつ見るので、計算量は O(N)。N が 10^7 を超え始めると 1 秒に間に合わなくなるので、そのときに二分探索やハッシュへの切り替えを考えます。

flowchart LR
  A0[A0] -->|一致?| A1[A1]
  A1 -->|一致?| A2[A2]
  A2 -->|一致?| A3[A3]
  A3 -->|一致?| A4[A4]
  A4 --> R{見つかった?}

Step 2: 「あるかどうか」と「位置」

Python なら X in A で存在判定 (内部は線形探索)、A.index(X) で位置取得 (見つからないと例外)。競プロでは自前ループの方がフラグ管理・早期 return が直感的です。

Step 3: 早期終了で平均を速くする

見つかった瞬間 break すれば、最悪は変わらないが平均的に半分の計算量で済みます。ループ後に else を付けると「最後まで break されなかった」場合の処理が書けます。

flowchart TB
  L[for x in A] --> C{x == X?}
  C -- Yes --> Y[Yes と出力して break]
  C -- No --> L
  L -.最後まで no_break.-> N[No と出力]

詰まりやすい点

  • 「ある」しか調べないなら早期 break を入れる。性能と読みやすさの両得。
  • N が 10^7 を超えるなら、線形探索ではなく 二分探索 (A11) や 辞書 (A54) に切り替えるサイン。
  • 入力に重複が含まれるかで、「最初に見つけた」「全部数える」の挙動が変わる。
