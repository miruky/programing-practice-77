法則 A22

すごろく型 DP — 前向き遷移で書く

自分の dp が決まったら、次の dp を更新する

「現在位置 i から、1 個進む / 2 個進む / ...」のように「自分から次に配る」 スタイルの DP は「前向き遷移」「push 型」と呼ばれます。

A16 のように 「自分が受け取る」 スタイル (後ろ向き遷移、pull 型) と数学的に等価ですが、「次の遷移先が複数あって、それぞれを更新する」場合 (DAG 上の最長路など) は前向きの方が自然に書けます (C16 など)。

Step 1: 全マスを INF で初期化

到達不能 = INF で初期化、dp[0] = 0 だけスタート地点。

flowchart LR
  S["dp[0] = 0"] --> X["dp[1..N-1] = INF"]

Step 2: 前向き遷移

dp[i] が確定したら、dp[i+1] と dp[i+2] を更新。

for i in range(N):
    if i+1 < N: dp[i+1] = min(dp[i+1], dp[i] + A[i])
    if i+2 < N: dp[i+2] = min(dp[i+2], dp[i] + B[i])

Step 3: 後ろ向きとの違い

後ろ向き (A16): dp[i] = min(dp[i-1]+A[i], dp[i-2]+B[i])。前向き (A22): dp[i+1] = min(dp[i+1], dp[i]+A[i])。どちらも同じ答えを返す。書きやすい方を選ぶ。

flowchart LR
  B["後ろ向き: 自分が貰う"] -.等価.- F["前向き: 自分が配る"]

詰まりやすい点

  • 前向き遷移は配列を min で更新する形になるため、初期化を INF にしておく ことが必須。
  • 添字のはみ出し (i+2 >= N) に注意。if で囲むか、配列を余分に取る。
  • グラフ上の DP (DAG 上最長路など) は前向きの方が書きやすい。
