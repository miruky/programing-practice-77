法則 A53

優先度付きキュー — 最小値 / 最大値を高速に取り出す

ヒープ。挿入と最小値削除が O(log N)

最小値 (または最大値) を効率的に取り出す ことに特化したデータ構造。二分ヒープが標準的な実装で、挿入と取り出しが共に O(log N)。

Python は heapq モジュール。最小ヒープ専用 なので、最大値が欲しいときは −1 倍して入れる トリックを使う。ダイクストラ (A64)、ジョブスケジューリング (B53) などで必須。

Step 1: 操作

  • push: heappush(h, x) — O(log N)
  • pop min: heappop(h) — O(log N)
  • peek min: h[0] — O(1)
flowchart LR
  P[heappush] --> H[ヒープ内部で並べ替え]
  H --> R["h[0] が常に最小"]

Step 2: 最大ヒープへの変換

Python は最小ヒープしかないので、−1 倍 して入れる。

heapq.heappush(h, -x)
max_x = -heapq.heappop(h)

Step 3: タプルで複合キー

(priority, payload) のタプルを入れると、優先度順にソートされる。ダイクストラの (距離, 頂点) ペアが典型例。

詰まりやすい点

  • heapq は 最小ヒープのみ。最大が必要なら -x で入れる。
  • h[1] は 2 番目に小さい値ではない (ヒープ構造のため)。
  • 既存のリストを heapify(h) でヒープ化すると O(N) (一個ずつ push より速い)。
