法則 A21

区間 DP — 区間 [l, r] を状態にする

小さい区間から大きい区間へ。LEN を外ループに置くのが定石

「両端から処理して中央に向かう」「最後に切る場所を全部試す」といった構造の問題で、区間 [l, r] を状態とする 2 次元 DP。

区間長 LEN を外側でループ するのが定石。小さい区間の答えが先に確定するように順序を制御する必要があるからです。回文 (B21)、切り方 (C18)、ブロック取り (A21) など応用の幅が広い。

Step 1: 状態の定義

dp[l][r] = 区間 [l, r] に対する最適値。「ブロック取りなら累積スコア」「回文なら最長回文部分列長」など、問題ごとに意味は変わる。

Step 2: ループの順序

区間長 LEN を外側、左端 l を内側。これで「短い区間の dp を先に確定」できる。

flowchart LR
  L1[LEN=1] --> L2[LEN=2]
  L2 --> L3[LEN=3]
  L3 --> LN[...]

Step 3: 遷移パターン

両端を 1 つずつ縮める / 中央で 2 分割する / 端を取り外すなど、「区間をどう分解するか」 で式が決まる。

ブロック取りなら dp[l][r] = max(dp[l-1][r] + score(l-1), dp[l][r+1] + score(r+1))。

flowchart LR
  D["dp[l][r]"] --> A["dp[l-1][r] + 左端の得点"]
  D --> B["dp[l][r+1] + 右端の得点"]

詰まりやすい点

  • 外ループは 必ず区間長 にする。l, r を独立にループすると dp[l+1][r-1] が未確定で参照失敗。
  • 中央で 2 分割するタイプ (C18) は内側の 分割点 k も全探索する 3 重ループ → O(N³)。
  • 1-indexed と 0-indexed の混在に注意。本サイトは入力配列にダミーを詰めて 1-indexed に揃える流儀。
