法則 A06

1 次元累積和 — 区間和を O(1) で答える

先に「先頭からの合計」を作っておけば、引き算で区間和が出る

数列 A[1..N] に対して、S[i] = A[1] + A[2] + ... + A[i] を前計算しておけば、任意の区間和 A[l] + ... + A[r] は S[r] - S[l-1] で O(1) に求まります。

「クエリの度に毎回足し算する O(Q · N)」を「前計算 O(N) + クエリ O(Q)」に圧縮する、競プロにおける最重要テクの 1 つ。2 次元 (A08) / いもす法 (A07) / セグメント木 (A58) など、ここを起点に多くの法則が広がっていきます。

Step 1: 番兵の 0 を置く

S[0] = 0 と定義しておくと、l=1 のような端のケースでもS[r] - S[l-1] = S[r] - S[0] で同じ式で書けます。Python の配列は 0-indexed ですが、S だけは 1-indexed に揃えると本の式と一致して読みやすい。

flowchart LR
  S0["S[0]=0"] --> S1["S[1]=A1"]
  S1 --> S2["S[2]=A1+A2"]
  S2 --> S3["S[3]=A1+A2+A3"]
  S3 --> Sn[...]

Step 2: 区間和は「右端 − 左手前」

A[l] + ... + A[r] = S[r] - S[l-1]。下の図のように、長いバーから短いバーを引いて差分を取るイメージで覚えると一発で覚えられます。

flowchart LR
  SR["S[r] = A1+...+Ar"] --> M["−"]
  SL["S[l-1] = A1+...+A(l-1)"] --> M
  M --> ANS["= Al + ... + Ar"]

Step 3: 半開区間で書く流儀

半開区間 [l, r) (= r を含まない) で考えると、S[r] - S[l] と式から -1 が消えてズレが減ります。セグメント木 (A58) など他の法則とも整合しやすく、慣れたらこちらの記法もおすすめ。

詰まりやすい点

  • S[l-1] の -1 は閉区間 [l, r] 用。 問題が 半開区間 で来たら S[r] - S[l] に書き換える。
  • l = 1 のとき S[0] を参照する。番兵を置いていないと IndexError。
  • 答えが大きくなる問題では int のオーバーフローを気にする必要はない (Python は任意精度)。
