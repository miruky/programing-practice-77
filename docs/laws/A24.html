法則 A24

LIS — 最長増加部分列

O(N log N) で部分列の最長を求める伝説のアルゴリズム

配列 A から、狭義に増加する 部分列 (連続でなくてよい) のうち最長のものの長さ。素朴な DP は O(N²) ですが、二分探索を使うと O(N log N) に。

「L[k] = 長さ k+1 の増加部分列の最後の値の最小値」という配列を作りながら走査する、情報を圧縮した DP の傑作。一見魔法に見えますが、L を保ち続けるだけ。

Step 1: L の意味

`L[k] = 長さ k+1 の増加部分列を作るときの『最後の値の最小値』。短い部分列ほど、最後の値が小さい方が後で伸ばしやすい。

flowchart LR
  A["A = [3, 1, 4, 1, 5, 9, 2, 6]"] --> L["L = [1, 2, 6, 9, ...]"]
  L --> M["L[k] = 長さ k+1 のときの最後の値 min"]

Step 2: 走査と更新

各 A[i] について、L の中で「A[i] が入る位置」を bisect_left で求める。

  • 末尾なら append (LIS が 1 伸びる)
  • そうでなければ L[pos] = A[i] で更新 (より小さい値で塗り替える)
flowchart TB
  X["A[i]"] --> P["pos = bisect_left(L, A[i])"]
  P --> J{pos == len L?}
  J -- Yes --> AP["L.append(A[i])"]
  J -- No --> UP["L[pos] = A[i]"]

Step 3: 答えは len(L)

走査終了時点での len(L) が LIS の長さ。L 自体は LIS そのものではない ことに注意 (各位置で違う LIS を「上書き」しているため)。

詰まりやすい点

  • L の中身は LIS の実体ではない。長さを取り出すだけで、復元には別途記録が必要。
  • 「真に増加」と「単調増加 (=同じ値も OK)」で bisect_left / bisect_right を使い分け。
  • 同じ長さでも様々な LIS がある。「いずれか 1 つを出力」なら復元時に最後の選択を保持。
