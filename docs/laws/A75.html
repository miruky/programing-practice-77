法則 A75

締切付きジョブ DP — ソート + DP

締切順に並べてから DP

「各ジョブに所要時間 T と締切 D がある。解けるジョブ数を最大化」という古典問題。まず締切で昇順ソート、それから DP が定石。

ソートしない素朴な DP では「順番」も状態に入れる必要があり爆発するが、ソートすれば順番が固定され、dp[i][j] = i 個目までで時刻 j のときの最大解ける数 で済む。

Step 1: 締切で昇順ソート

「締切が早いものから処理を考える」ように並べておく。貪欲ではなく DP のための前処理 であることに注意。

flowchart LR
  J[ジョブ群] --> S[締切順ソート]

Step 2: DP の遷移

dp[i+1][j] = max(dp[i][j], dp[i][j-t] + 1) (時刻 j に i を解くなら)。 条件は t ≤ j ≤ d (時刻 j に間に合い、ジョブ i の所要時間が j 以下)。

flowchart LR
  D1["dp[i][j] (i を解かない)"] --> M[max]
  D2["dp[i][j-t] + 1 (i を解く)"] --> M
  M --> D["dp[i+1][j]"]

Step 3: 答えの取り出し

max(dp[N]) = どの時刻でもいいので最大解けるジョブ数。

詰まりやすい点

  • ソートを忘れると遷移が壊れる。前処理が本質。
  • 時刻の最大値 (MAX_D) が大きいとメモリ・計算量が辛い。
  • 「給料最大化」など別最適化なら別 DP が必要 (B39)。
