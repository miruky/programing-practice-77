答えの候補を一直線に並べ、NG と OK の境目を探します。毎回、真ん中を見て半分捨てます。

  1. 何が OK で何が NG かを言葉で決める。
  2. left / right が常に条件を満たすように初期化する。
  3. mid の判定結果で、境目がどちら側にあるかだけ更新する。

詰まりやすい点: 探索しているのは値そのものではなく、条件が切り替わる境目です。

# =====================================================================
# A15: 座標圧縮
# ---------------------------------------------------------------------
# 【概要】配列 A の各要素について、A 全体での "順位" (1 始まり) を求める。
# 【アルゴリズム】座標圧縮
#   1. A から重複を取り除いてソートした配列 T を作る。
#   2. T 内での A[i] の位置 (二分探索) +1 がその順位となる。
# 【計算量】O(N log N)
# 【用途】後段でセグメント木・BIT を使う前処理として頻出。
# =====================================================================
# ---------------------------------------------------------------------
# 【読み方の地図】
# - 問題の型: 座標圧縮 / 二分探索 / 難易度目安: ★2
# - 要約: 重複を除いてソートし、各値の順位に置き換えるテクニック。
# - まず入力ブロックで、どの変数・配列に何が入るかを確認する。
# - 次に中心となる配列・状態・データ構造の意味を 1 行で言語化する。
# - 最後に答えを出す式や出力ループだけを、入力例と照らして追う。
# 【このコードで特に見る場所】
# - 二分探索は値を探すというより、条件が NG から OK に切り替わる境目を探す。
# - bisect_left は「その値を入れてもソートが崩れない一番左の位置」を返す。
# - LIS や座標圧縮では、配列そのものではなく順位・置き場所を管理する。
# ---------------------------------------------------------------------

import bisect

# --- 入力 -----------------------------------------------------------
N = int(input())
A = list(map(int, input().split()))

# --- 重複を除去してソート -----------------------------------------
# set() で重複除去、その後 sort() で昇順に並べる。
T = list(set(A))
T.sort()

# --- 各値の順位を二分探索で求める ---------------------------------
B = [None] * N
for i in range(N):
    B[i] = bisect.bisect_left(T, A[i])  # T 内での位置 (0-indexed)
    B[i] += 1                           # 1-indexed に直す

# --- 空白区切りで出力 ------------------------------------------------
Answer = [str(i) for i in B]
print(" ".join(Answer))
