B41: Reverse of Euclid (A41 応用 ─ 互除法の逆操作)

概要

(1,1) からスタートして「片方に他方を足す」を繰り返し、最終的に (X, Y) になる過程を 1 つ出力する。X, Y は互いに素であることが 保証される。

アルゴリズム

(X, Y) から逆向きに操作を辿る ─ つまり「大きい方から小さい 方を引く」を、両方が 1 になるまで繰り返す。これは GCD 互除法を引き算ベースで実行しているのと同じ。

計算量

O(log max(X, Y))。

読み方の地図

  • 問題の型: 数学 / GCD / シミュレーション / 難易度目安: ★3
  • 要約: ユークリッド互除法を逆向きに辿り、(1,1) から (X,Y) までの過程を出力する。
  • まず入力ブロックで、どの変数・配列に何が入るかを確認する。
  • 次に中心となる配列・状態・データ構造の意味を 1 行で言語化する。
  • 最後に答えを出す式や出力ループだけを、入力例と照らして追う。

このコードで特に見る場所

  • 全探索ではなく、式・余り・周期・不変量に置き換えて一気に答えるタイプ。
  • 小さい例で 1 回手計算すると、なぜその式になるかがかなり見えやすい。
  • mod が出る問題では、足す・引く・掛けるたびに余りを取って値を小さく保つ。

この法則をもっと詳しく A41: 鳩ノ巣原理 — 「必ず同じ箱に 2 つ入る」 → 
# --- 入力 -----------------------------------------------------------
X, Y = map(int, input().split())

# --- 逆向きシミュレーション ----------------------------------------
Answer = []
while X >= 2 or Y >= 2:
    Answer.append([X, Y])
    if X > Y:
        X -= Y                            # 大きい方から小さい方を引く
    else:
        Y -= X
Answer.reverse()                          # 順方向に直す

# --- 出力 -----------------------------------------------------------
print(len(Answer))
for i in range(len(Answer)):
    print(str(Answer[i][0]) + ' ' + str(Answer[i][1]))
