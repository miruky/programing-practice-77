基本問題 A16: ダンジョン1 1 次元 DP の基本問題。最小コストで部屋 N にたどり着く。

B16: Frog 1 (A16 応用 / EDPC-A の典型 DP)

概要

足場 0..N-1 の高さ H[i] が与えられる。0 から N-1 まで、1 個 or 2 個先のマスに飛び移れるとき、移動コストの最小値を求める。 コストは |H[i] - H[j]|。

アルゴリズム

古典 1 次元 DP。dp[i] = 足場 i に到達する最小コスト。 dp[i] = min(dp[i-1] + |H[i-1] - H[i]|, dp[i-2] + |H[i-2] - H[i]|)

計算量

O(N)。

読み方の地図

  • 問題の型: DP / 難易度目安: ★3
  • 要約: 1 個 or 2 個先に飛ぶカエル DP の古典問題 (EDPC-A)。
  • まず入力ブロックで、どの変数・配列に何が入るかを確認する。

  • 最後に答えを出す式や出力ループだけを、入力例と照らして追う。

コメント表示中だけの読み方ガイド 動的計画法

dp は「途中結果のメモ」です。dp[i] や dp[i][j] の日本語意味を決めると、遷移式が読みやすくなります。

  1. dp の添字が何を表すかを先に固定する。
  2. 初期状態だけを手で入れる。
  3. 小さい状態から大きい状態へ、使う/使わない・進む/進まないで遷移する。

詰まりやすい点: 式を先に暗記するより、dp[i] の意味を日本語で読めるかを優先します。

難易度目安: ★3DP
この法則をもっと詳しく A16: 1 次元 DP の基本 — dp[i] の意味を 1 行で言える →
小さい状態から大きい状態へ
dp[i-2]dp[i-1]dp[i]
# --- 入力 -----------------------------------------------------------
N = int(input())
H = list(map(int, input().split()))

# --- DP 配列の初期化 ------------------------------------------------
dp = [None] * N

# --- 動的計画法 -----------------------------------------------------
# 0 番目はスタートなのでコスト 0、1 番目は必ず 1 個ジャンプして来る。
dp[0] = 0
dp[1] = abs(H[0] - H[1])
for i in range(2, N):
    # 1 個前から来るか、2 個前から来るかの安い方を取る。
    dp[i] = min(dp[i - 1] + abs(H[i - 1] - H[i]),
                dp[i - 2] + abs(H[i - 2] - H[i]))

# --- 出力 -----------------------------------------------------------
print(dp[N - 1])
