dp は「途中結果のメモ」です。dp[i] や dp[i][j] の日本語意味を決めると、遷移式が読みやすくなります。

  1. dp の添字が何を表すかを先に固定する。
  2. 初期状態だけを手で入れる。
  3. 小さい状態から大きい状態へ、使う/使わない・進む/進まないで遷移する。

詰まりやすい点: 式を先に暗記するより、dp[i] の意味を日本語で読めるかを優先します。

# =====================================================================
# B19: Knapsack 2 (A19 応用 ─ 価値添字のナップサック DP)
# ---------------------------------------------------------------------
# 【概要】N 個の品物 (重み w[i], 価値 v[i]) と容量 W のナップサックで、
#         価値の総和を最大化する。W は最大 10⁹ なので、通常の重み添字 DP
#         (O(NW)) では間に合わない。
# 【アルゴリズム】価値の合計を添字に取り、最小重みを管理する DP に切り替える。
#                 dp[i][j] = 「i 個目まで見て合計価値 j を達成するときの最小重さ」
#                 → 最後に dp[N][j] ≦ W となる最大 j を答えとして出力。
# 【計算量】O(N · sum(v))。v_i ≦ 1000 なら 10⁵ 程度に抑えられる。
# =====================================================================
# ---------------------------------------------------------------------
# 【読み方の地図】
# - 問題の型: DP / ナップサック / 難易度目安: ★4
# - 要約: 価値を添字とする別解のナップサック DP。重量と価値の役割を入れ替える。
# - まず入力ブロックで、どの変数・配列に何が入るかを確認する。
# - 次に中心となる配列・状態・データ構造の意味を 1 行で言語化する。
# - 最後に答えを出す式や出力ループだけを、入力例と照らして追う。
# 【このコードで特に見る場所】
# - dp の添字が何を意味するかを最初に日本語で固定する。ここが一番重要。
# - 初期状態は「何も選んでいない」「スタート地点」など、手でわかる状態だけ入れる。
# - 遷移は「使う/使わない」「前から来る/上から来る」など、直前の状態から読む。
# ---------------------------------------------------------------------

# --- 入力 -----------------------------------------------------------
N, W = map(int, input().split())
w = [None] * (N + 1)
v = [None] * (N + 1)
for i in range(1, N + 1):
    w[i], v[i] = map(int, input().split())

# --- DP テーブル (添字: 価値, 値: 重みの最小) ----------------------
INF = 10 ** 15
dp = [[INF] * 100001 for _ in range(N + 1)]
dp[0][0] = 0
for i in range(1, N + 1):
    for j in range(100001):
        if j < v[i]:
            dp[i][j] = dp[i - 1][j]
        else:
            # 取らない / 取って重み w[i] を加算 の min を取る。
            dp[i][j] = min(dp[i - 1][j], dp[i - 1][j - v[i]] + w[i])

# --- W を超えない範囲で達成できる最大の価値 j を探す ---------------
Answer = 0
for j in range(100001):
    if dp[N][j] <= W:
        Answer = j
print(Answer)
